This skill guides the creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. You must implement real working code with exceptional attention to aesthetic details, micro-interactions, and creative choices.

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme aesthetic direction (e.g., brutally minimal, maximalist chaos, retro-futuristic, elegant/luxury, organic/natural, brutalist/raw, art deco, soft/pastel, industrial, glassmorphism, etc.). Find an inspiration point and commit to it fully.
- **Constraints**: Technical requirements (framework, performance, accessibility).
- **Differentiation**: What makes this UI unforgettable? What's the one hallmark design feature?

**CRITICAL**: Choose a clear conceptual direction and execute it with mathematical precision. Bold maximalism and refined minimalism both work — intentionality is key, not intensity.

When implementing code (HTML/CSS/JS, React, Vue, Next.js, etc.), ensure it is:
1. Production-grade and fully functional.
2. Visually striking and inherently memorable.
3. Cohesive with a firm point-of-view matching the chosen aesthetic.
4. Meticulously refined down to the last pixel.

## Frontend Aesthetics Guidelines

Pay particular attention to these areas:
- **Typography**: Choose fonts that are beautiful, unique, and characterful. **Avoid generic fonts** at all costs (Arial, Inter, Helvetica, Roboto, System defaults). Instead, opt for distinctive font pairings (e.g., a bold display font like `Outfit` or `Space Grotesk` combined with a refined body font like `Lora` or `DM Sans`). Use Google Fonts freely.
- **Color & Theme**: Commit to a cohesive and striking color narrative. Use precise CSS variables. A color palette dominated by one or two sharp, distinct accents performs significantly better than a timid, evenly distributed set of generic colors.
- **Motion & Interaction**: Use micro-interactions and animations to bring the application to life. Prioritize CSS-only transitions for pure HTML or robust libraries (Framer Motion, GSAP) if using frameworks.
  - Implement cohesive staggered reveals on page load (`animation-delay`).
  - Introduce delightful hover states, unexpected scroll-triggered animations.
- **Spatial Composition**: Break out of standard boxes. Use unexpected layouts, asymmetry, smart overlapping, diagonal flows, or grid-breaking elements. Deploy generous negative space OR heavily controlled density depending on the tone.
- **Backgrounds & Atmosphere**: Create deep, immersive environments. Never default to plain flat background colors. Introduce subtle noise textures, layered gradients, geometric mesh backgrounds, glassmorphism layers, or dramatic shadows.

## Anti-Patterns (What NOT to do)
NEVER use generic AI-generated templates:
- ❌ Overused font families (Inter, Roboto, Arial).
- ❌ Cliché color schemes (like plain purple gradients over white backgrounds).
- ❌ Boxy, predictable layouts lacking depth or context.
- ❌ Cookie-cutter Bootstrap/Tailwind standard defaults.

## Complexity Matching
Match the implementation code complexity to the aesthetic vision. Maximalist designs demand extensive animations, overlays, and effects. Minimalist ones require absolute surgical restraint—obsessive attention to spacing, contrast, typography scaling, and subtle borders. 

> Remember: Don't hold back. Execute creative, outside-the-box frontend tasks with maximum polish.
